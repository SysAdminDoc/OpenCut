## 1. Identity

OpenCut is a **local-first, MIT-licensed automation backend for Adobe Premiere Pro** (plus a DaVinci Resolve scripting bridge). It ships:

- A Python/Flask server bound to `127.0.0.1:5679` (HTTP) + `:5680` (WebSocket) + `:5681` (MCP JSON-RPC).
- Two Premiere panels: a **CEP** panel (`com.opencut.panel`, ~8500-line `main.js`) for Premiere 2019+ and a **UXP** panel (`com.opencut.uxp`, ~1700-line ES module) for Premiere 25.6+ — same backend, near-feature-parity.
- An MCP sidecar (`opencut.mcp_server`) exposing 27+ tools to MCP clients (Claude Code, Cursor, etc.).
- A WPF .NET 9 installer + legacy Inno Setup, Docker (CPU + GPU compose), PyInstaller exe spec.
- Optional pip extras (`[standard]`, `[ai]`, `[ai-gpu]`, `[music]`, `[diarize]`, `[mcp]`, `[all]`, `[dev]`, …) — only `flask + click + rich + flask-cors + python-json-logger + psutil` are required.

Marketing identity: "free, open-source Premiere Pro extension that brings AI-powered video editing automation … all running locally on your machine. No subscriptions, no cloud, no API keys required."

## 3. Architecture

```
Premiere CEP panel ─┐
Premiere UXP panel ─┼─► HTTP localhost:5679  ─► Flask app (create_app() factory)
DaVinci Resolve ────┤   WebSocket :5680           ├─ routes/* (101 blueprints) ─► core/* (523 modules) ─► FFmpeg / Whisper / Demucs / Torch / ONNX
MCP client     ─────┘   MCP HTTP :5681            ├─ jobs.py (async @async_job decorator, SQLite persistence, GPU rate limit)
                                                  ├─ security.py (CSRF, path-traversal, SSRF guards, safe_pip_install)
                                                  ├─ auth.py (loopback bypass + 256-bit token for non-loopback binds)
                                                  ├─ registry.py (feature readiness states for UI gating)
                                                  ├─ model_cards.py + opencut/_generated/* (license/model truth)
                                                  └─ checks.py (35+ check_X_available() gates)
```

Highlights:
- `@async_job("type")` decorator is the **only** sanctioned way to write long-running routes — it handles thread spawn, job_id correlation, cancellation, SQLite persistence (`~/.opencut/jobs.db`), structured error mapping, and rate-limit acquire/release pairing.
- 101 blueprints registered through a **deterministic ordered tuple** in `routes/__init__.py`; runtime route-collision guard refuses startup on duplicate `(method, path)` pairs.
- **User data store at `~/.opencut/`** (jobs.db, footage_index.db, social_credentials.json, llm_settings.json, brand_kit.json, onboarding.json, plugin/lock files, model caches…) — all access through `user_data.py` wrappers with per-file locks and atomic `os.replace` writes.
- **Plugin system** validates a v1 manifest (F116 shipped) with `plugin.lock.json` hash, `OPENCUT_PLUGIN_ALLOW_UNSIGNED` opt-in, and a `SUPPORTED_CAPABILITIES` allowlist before mounting any blueprint.
- **Wave architecture**: code is layered by ship date — `wave_a_routes.py` … `wave_l_routes.py` add new feature surfaces atop the original blueprint set, each with its own `check_*_available()` gates. The F-numbered features (F001-F120) are the newer governance/infra layer, separate from the wave-letter (A-T) AI-model layer.

## 9. Security posture (snapshot)

- **Auth**: localhost bypass; 256-bit token at `~/.opencut/auth.json` (POSIX `0600`) required when `OPENCUT_ALLOW_REMOTE=1`. CSRF token in `X-OpenCut-Token` on all POST/DELETE (F069/F112).
- **Path containment**: `validate_path()` / `validate_filepath()` with realpath + prefix whitelist + null-byte / `..` / UNC checks. The 2026-05-17 dirty diff adds the UNC-post-realpath guard.
- **SSRF**: `_validate_webhook_url()` / `_validate_download_url()` block localhost / loopback / private / link-local / reserved. Offline-safe (no DNS resolution required for public hostnames).
- **Plugin sandbox**: F116 manifest v1, `plugin.lock.json` hash, capability allowlist, `OPENCUT_PLUGIN_ALLOW_UNSIGNED` opt-in.
- **`/file` preview allowlist**: GET-only, MIME-checked, resolved path must be under `tempfile.gettempdir()` or `~/.opencut`.
- **`/system/open-path`**: blocks executable extensions in `open` mode; `reveal` mode (Explorer/Finder) unrestricted.
- **SBOM**: `scripts/sbom.py` regenerates `requirements-lock.txt`; `pip-audit` clean as of last F094 run. `deep-translator` (PYSEC-2022-252) removed.
- **Node advisories**: `esbuild` pinned `^0.25` via `overrides` to close GHSA-67mh-4wv8-2f99. Vite `.map` traversal CVE explicitly waived in `docs/NODE_ADVISORIES.md` because we never run `vite dev/preview` in production.
- **Concurrency**: `apply_config()` under `job_lock` to prevent TOCTOU. All `Popen.wait()` calls bounded (`timeout=1800` on Demucs). Atomic `os.replace` for all user-data writes.
- **PHI/PII**: scrubbed in `core/issue_report.py` (HOME → `~`) and `core/crash_packet.py` (same `_scrub_paths` helper).
- **Provenance**: F110 C2PA sidecars on export, Ed25519 signing when `OPENCUT_C2PA_SIGNING_KEY` + `cryptography` present.

No known unpatched CVEs as of last `release_smoke.py` run. The dirty diff closes two **defence-in-depth** holes (UNC bypass via symlink, full `127.0.0.0/8` instead of just `127.0.0.1`).
